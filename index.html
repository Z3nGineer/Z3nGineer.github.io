Linux Systems Engineering

CachyOS / Arch — Bare Metal

My daily driver is a custom-tuned Arch Linux (CachyOS) workstation. Both the CPU and GPU are fully overclocked and stability-tested — not just XMP and call it a day. Every layer is configured by hand — kernel scheduler, power limits, fan curves, sleep state management, pacman hooks. It breaks sometimes. I fix it. That's the deal.

GPU Overclock (RTX 5070 Ti) — +400 core, +5000 memory, 350W power limit, custom fan curve tuned for noise/thermals. Rock solid under sustained ML training and gaming
CPU Overclock (Ryzen 9 9800X3D) — Precision Boost override, per-CCX tuning, curve optimizer. Hours of stress testing to dial it in. Both overclocks done by hand, no presets
OS & Kernel — CachyOS (Arch), sched-ext bpfland scheduler, custom pacman hooks, modprobe configs, multi-monitor persistence. Previously ran the same hardware tuned on Windows
ML Stack — CUDA 12.8, PyTorch nightly cu128, separate venvs for app vs ML workloads, Docker services
Gaming — Proton + DXVK on Linux, per-game launch configs, DX11 over DX12 for lower overhead, multi-monitor Wayland

HeatSync

Real-Time System Monitor

A cross-platform desktop system monitor I built and open-sourced. CPU and GPU temps, usage, VRAM, clock speeds, power draw — everything at a glance. Built it because I got tired of opening five different apps to check if my GPU was throttling. Runs natively on Linux, Windows, and macOS — which is rare for a tool this simple. There's a GitHub Actions workflow that automatically builds all three installers (AppImage, .exe, .dmg) on every release. Full details on the HeatSync page.

Live Telemetry — CPU, GPU, temps, usage with real-time sparkline graphs
True Cross-Platform — Linux AppImage, Windows .exe, macOS .dmg — all built automatically via GitHub Actions CI
Minimal Footprint — Sits at the top of the screen, stays out of the way
